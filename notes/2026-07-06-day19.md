
# Exp058 — Six-Language Indexed Baseline

## Objective
Validate end-to-end multilingual Whisper Small + LoRA training using the indexed ANV and Swahili dataset infrastructure developed in Exp056 and Exp057.

## Configuration
- Model: Whisper Small
- Training: LoRA fine-tuning
- Languages: Kikuyu, Luo, Kalenjin, Maasai, Somali, Swahili
- Balanced dataset:
  - Train: 80 samples/language (480 total)
  - Eval: 20 samples/language (120 total)
- Audio loading:
  - ANV datasets resolved via global Parquet index
  - Swahili resolved via TAR index
- Runtime: Google Colab L4 GPU

## Results
- Training completed successfully.
- End-to-end multilingual pipeline validated.
- Swahili TAR index resolved audio correctly.
- No resolver failures during training.
- Stable LoRA training and evaluation completed successfully.

## Key Findings
- Indexed audio resolution works correctly for all six languages.
- GPU utilization is limited by data loading rather than model computation.
- Swahili TAR extraction remains the primary runtime bottleneck.
- Current architecture is functionally correct but not yet optimized for throughput.

## Decision
Exp058 establishes the first fully working multilingual training pipeline.

Next experiment (Exp059) will cache decoded audio locally before training to eliminate repeated TAR extraction and significantly reduce experiment runtime.

# Exp059 — Prep

## Objective
Align internal evaluation with the Kaggle scoring method and prepare a larger balanced six-language training run.

## Why
The competition leaderboard uses the unweighted mean WER across the six target languages. Therefore, every experiment from Exp059 onward should report:
- global WER
- per-language WER
- Kaggle-style average WER

## Competition Metric
Average WER = mean of:
- WER_swa
- WER_kik
- WER_luo
- WER_som
- WER_mas
- WER_kln

Each language contributes equally, regardless of dataset size.

## Exp059 Plan
- Build a larger balanced multilingual dataset.
- Keep all six languages equally represented.
- Add per-language WER reporting.
- Use cached audio if needed to reduce data-loading bottlenecks.
- Compare against Exp058.

## Proposed Dataset
Initial target:
- Train: 500 samples per language
- Eval: 100 samples per language
- Total train: 3,000
- Total eval: 600

## Success Criteria
- Dataset builds with all six languages balanced.
- Evaluation reports per-language WER.
- Evaluation reports Kaggle-style mean WER.
- Training completes without resolver failures.
- Runtime is measured against Exp058.

## Exp059 Progress Update

### Major Design Decision

Exp059 will evolve the current training script into a reusable experiment platform rather than a single-purpose training script.

### Rationale

Previous experiments focused on validating infrastructure.

From Exp060 onward, the primary objective shifts toward systematic WER reduction.

To support rapid experimentation, the training pipeline will become configuration-driven.

### Configuration Philosophy

The YAML configuration should describe **what experiment is being run**, while the Python code describes **how the experiment is executed**.

Future experiments should primarily require configuration changes rather than Python modifications.
