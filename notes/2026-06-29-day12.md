## Experiment 036A — AfriVoices Dataset Profiling

### Objective

Build a reusable dataset profiler to inspect the complete AfriVoices competition corpus before creating a unified training manifest.

### Major Accomplishments

- Successfully authenticated and accessed all gated Hugging Face datasets.
- Implemented a six-language profiler supporting seven source repositories.
- Successfully profiled:
  - Swahili
  - Kikuyu
  - Luo (Dholuo)
  - Somali
  - Maasai
  - Kalenjin
- Generated:
  - afrivoices_profile.md
  - afrivoices_language_summary.csv
  - afrivoices_split_summary.csv
  - afrivoices_top_dialects.csv
  - visualization figures
- Produced a reusable profiling pipeline for future datasets.

### Dataset Statistics

- Total utterances: 1,081,999
- Total measured audio: 5,542.97 hours
- Languages: 6
- Speakers detected: 3,577

Language distribution:

| Language | Hours |
|----------|------:|
| Swahili | 3201.18 |
| Luo | 681.70 |
| Kalenjin | 496.26 |
| Maasai | 482.67 |
| Somali | 478.19 |
| Kikuyu | 202.97 |

### Key Findings

- The corpus is highly imbalanced, with Swahili contributing the majority of training hours.
- Kaggle evaluates languages equally despite the imbalance, making future language-balancing experiments potentially valuable.
- Scripted and unscripted speech distributions differ substantially between languages.
- Swahili speaker metadata appears incomplete in the current parser implementation and should be revisited later if speaker-level analyses become important.
- The profiler successfully validates that all competition datasets are accessible and ingestible.

### Outcome

Experiment 036A successfully completed.

The project is now ready to build unified training manifests in Exp036B before beginning the first full AfriVoices training run.

## Exp037 Architecture Review — First Real AfriVoices Training Run

### Purpose

Exp037 will be the first training run using the real AfriVoices competition data rather than proxy data.

The purpose is not immediate leaderboard optimization. The purpose is to prove the full real-data training path:

manifest → audio resolution → preprocessing → training → validation → checkpoint

### Recommended Configuration

- Base model: openai/whisper-large-v3
- Training method: LoRA
- Hyperparameter baseline: Exp026 champion configuration
- Dataset: unified AfriVoices manifests from Exp036B
- Scope: filtered, capped multilingual subset

### Data Policy

Filter out:

- rows with missing or empty transcriptions
- rows with missing audio references
- rows with duration <= 0
- rows longer than 30 seconds

Use a balanced subset:

- up to 10,000 train rows per language
- up to 1,000 dev rows per language

This creates approximately:

- 60,000 training examples
- 6,000 validation examples

### Rationale

The full manifest contains over one million rows. Exp037 should validate the new training pipeline before spending significant GPU time.

A balanced subset prevents Swahili from dominating the first real-data run and aligns better with Kaggle's unweighted average WER across six languages.

### Required Engineering

Exp037 needs a real audio resolver:

- ANV-KE audio is stored as bytes inside parquet shards.
- Swahili audio is referenced through Swahili manifest/tar-style repository paths.
- The training dataset must resolve audio into arrays that Whisper can process.

### Evaluation Requirements

Exp037 should report:

- overall WER
- overall CER
- per-language WER
- per-language CER
- training loss
- validation loss
- learning-rate curve
- runtime
- checkpoint size

### Success Criteria

Exp037 is successful if:

- the unified manifests load correctly
- bad rows are filtered
- all six languages are represented
- audio references resolve
- training starts and completes
- validation metrics are produced
- a checkpoint is saved
