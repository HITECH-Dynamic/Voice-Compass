## Exp041 Plan Revision — Incremental ANV Indexing

### Reason for Revision

The original Exp041 description implied building a complete ANV audio index immediately.

Day 12 showed that full-corpus ANV audio indexing can be expensive and fragile because it requires downloading many large Parquet audio shards. A full indexing run may stall or consume excessive Colab runtime.

### Revised Plan

Exp041 will be split into staged work:

- Exp041A — Inspect whether a complete ANV index can be derived from repository metadata or manifest metadata without downloading every audio shard.
- Exp041B — Build partial/full indexes incrementally by language, split, and speech type.
- Exp041C — Validate index coverage against the unified training manifest.
- Exp042 — Integrate the best available index into the production dataset loader and benchmark speed.

### Updated Success Criteria

Exp041 is successful if we establish a scalable indexing strategy, not necessarily if the full index is completed in one run.

## Exp041A Results — ANV Dataset Index Strategy Inspection

### Objective

Determine whether transcript metadata contains sufficient information to locate audio files without scanning every Parquet shard.

### Findings

The answer is **no**.

Across all ANV repositories:

- Transcript CSVs contain `mediaPathId` references (for example, `VOICE_COLLECTION/...wav` or `PROMPT_COLLECTION/...wav`).
- Audio is distributed across many Parquet shards for each split.
- No transcript field identifies which Parquet shard contains a given recording.

Although repository inspection reveals how many Parquet shards exist for each split, there is no deterministic filename → shard mapping available from transcript or metadata files.

### Conclusion

Searching Parquet shards on demand is inefficient and does not scale.

The correct long-term solution is to build a persistent global lookup table:

mediaPathId
        ↓
Parquet shard

Once built, future preprocessing and training can resolve audio directly instead of scanning many large Parquet files.

### Outcome

✅ Exp041A completed successfully.

Research question answered.
