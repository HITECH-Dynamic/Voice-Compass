## Competition Evaluation Update

The Kaggle leaderboard uses unweighted average WER across the six target languages:

- Swahili: swa
- Kikuyu: kik
- Luo: luo
- Somali: som
- Maasai: mas
- Kalenjin: kln

This means the final score gives each language equal importance regardless of dataset size.

Implication:
Natural data distribution may overweight Swahili during training, but leaderboard scoring will not. Therefore, language-balanced sampling and per-language WER analysis should become high-priority Phase 4/5 experiments after the first competition baseline.

Submission format requires exactly three columns:

id,language,prediction

Language values must use ISO 639-3 codes, not full language names.

Edge deployment constraint:
Final model should target:
- RAM <= 8 GB during inference
- model size <= 1B parameters
- CPU-only inference
- near real-time transcription
- offline capability preferred

Note:
Whisper Large-v3 remains useful for establishing a strong research baseline, but final competition deployment may require model compression, distillation, quantization, or a smaller Whisper/XLS-R model.
