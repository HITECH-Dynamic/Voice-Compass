# Day 5 — Phase 4 Scaling

## Experiment 016 Plan

Purpose:

Test whether increasing model size from Whisper Medium to Whisper Large-v3 improves WER.

Configuration:

- Model: Whisper Large-v3
- Dataset: FLEURS Swahili
- Train examples: 3070
- Validation examples: 100
- Steps: 1000
- Learning rate: 2e-5
- Seed: 42
- Hardware: Google Colab L4 or better

Only major variable changed:

- Model changed from Whisper Medium to Whisper Large-v3

Reference baseline:

- Experiment 013: Whisper Medium, seed 42, best WER 0.1893

Question:

Does model scaling improve beyond the current champion?


## Experiment 016 Outcome

Purpose:

Test full fine-tuning of Whisper Large-v3 using the winning Experiment 013 recipe.

Configuration attempted:

- Model: Whisper Large-v3
- Dataset: FLEURS Swahili
- Train examples: 3070
- Validation examples: 100
- Steps: 1000
- Learning rate: 2e-5
- Seed: 42

Outcome:

Experiment 016 did not complete.

Failure modes observed:

1. FP16 full fine-tuning with batch size 4 failed with CUDA out-of-memory on L4.
2. Batch size 2 with gradient accumulation 2 still produced FP16 gradient scaling errors.
3. BF16 with batch size 2 still exceeded L4 memory.
4. BF16 with batch size 1 and gradient accumulation 4 started training but produced exploding losses and failed during evaluation with a dtype mismatch.

Observation:

Full fine-tuning Whisper Large-v3 is not practical on the available L4 environment using the current script.

Conclusion:

Phase 4 scaling cannot proceed through full fine-tuning Large-v3 on L4. The next scaling path should use parameter-efficient fine-tuning such as LoRA.
