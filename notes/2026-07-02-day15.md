# Day 15 — 2026-07-02

# Exp048 — Prep

## Objective
Run the first true multilingual AfriVoices baseline using the optimized cached training pipeline.

## Experiment
Exp048 — 5-language parquet multilingual baseline

## Languages
- Kikuyu: `kik`
- Kalenjin: `kln`
- Luo: `luo`
- Maasai: `mas`
- Somali: `som`

## Excluded
- Swahili: `swa`

## Reason for Excluding Swahili
Swahili currently uses `swahili_tar_ref`, which requires a separate tar-cache strategy. Exp048 focuses only on cache-friendly `parquet_bytes` languages.

## Control
- Same model: Whisper Small
- Same LoRA configuration
- Same cached ANV parquet loader
- Same W&B logging
- Same WER evaluation
- Same 300-step budget as Exp047

## Variable
- Dataset changes from Kikuyu-only to 5-language multilingual parquet data.

## Success Criteria
- Dataset builds successfully.
- Training completes successfully.
- W&B logs the run.
- WER is reported.
- Runtime remains practical.


# Exp048 — Decode Filter Patch

## Issue
Exp048 started successfully but failed during evaluation because one multilingual audio row could not be decoded.

## Error Pattern
- `LibsndfileError: Format not recognised`
- `EOFError`

## Fix
Add a reusable decode-filter script that tests each dataset row before training and writes clean train/eval parquet files.
