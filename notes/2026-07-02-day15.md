## Objective
Run the first true multilingual AfriVoices baseline using the optimized cached training pipeline.

## Excluded

Swahili currently uses `swahili_tar_ref`, which requires a separate tar-cache strategy. Exp048 focuses only on cache-friendly `parquet_bytes` languages.

# Exp048 — Decode Filter Patch

## Issue
Exp048 started successfully but failed during evaluation because one multilingual audio row could not be decoded.

## Error Pattern
- `LibsndfileError: Format not recognised`
- `EOFError`

## Fix
Add a reusable decode-filter script that tests each dataset row before training and writes clean train/eval parquet files.

## Findings

The multilingual model successfully trained across five languages without degrading overall recognition performance.

The cache-aware dataset loader remained stable.

The decode-filter utility successfully prevented corrupted audio from interrupting training.

The largest runtime cost was evaluation frequency rather than training itself.

## Conclusion

Exp048 establishes the first stable multilingual competition baseline.

Future experiments should reduce evaluation frequency while keeping multilingual training unchanged.
