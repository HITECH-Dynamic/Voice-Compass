# Exp048 — Prep

## Objective
Run the first true multilingual AfriVoices baseline using the optimized cached training pipeline.

## Experiment
Exp048 — 5-language parquet multilingual baseline

## Languages
- Kikuyu: `kik`
- Kalenjin: `kln`
- Luo: `luo`
- Maasai: `mas`
- Somali: `som`

## Excluded
- Swahili: `swa`

## Reason for Excluding Swahili
Swahili currently uses `swahili_tar_ref`, which requires a separate tar-cache strategy. Exp048 focuses only on cache-friendly `parquet_bytes` languages.

## Control
- Same model: Whisper Small
- Same LoRA configuration
- Same cached ANV parquet loader
- Same W&B logging
- Same WER evaluation
- Same 300-step budget as Exp047

## Variable
- Dataset changes from Kikuyu-only to 5-language multilingual parquet data.

## Success Criteria
- Dataset builds successfully.
- Training completes successfully.
- W&B logs the run.
- WER is reported.
- Runtime remains practical.


# Exp048 — Decode Filter Patch

## Issue
Exp048 started successfully but failed during evaluation because one multilingual audio row could not be decoded.

## Error Pattern
- `LibsndfileError: Format not recognised`
- `EOFError`

## Fix
Add a reusable decode-filter script that tests each dataset row before training and writes clean train/eval parquet files.


# Exp048 — Results

## Objective
Train the first true multilingual AfriVoices baseline using five cache-friendly languages.

## Dataset
Languages:
- Kikuyu (kik)
- Kalenjin (kln)
- Luo (luo)
- Maasai (mas)
- Somali (som)

Training rows: 396
Evaluation rows: 98

## Runtime Issue Discovered
Initial run failed due to six corrupted audio files.

Solution:
Created a reusable decode-filter utility that removed invalid audio before training.

Bad files removed:
- Train: 4
- Eval: 2

## Final Metrics

Training Steps:
500

Epochs:
2.53

Training Loss:
3.5893

Evaluation Loss:
3.2587

WER:
1.0028

Training Runtime:
3548.8 seconds (~59.1 minutes)

## Comparison

Exp047
WER: 1.0492

Exp048
WER: 1.0028

Relative WER improvement:
≈4.4%

## Findings

The multilingual model successfully trained across five languages without degrading overall recognition performance.

The cache-aware dataset loader remained stable.

The decode-filter utility successfully prevented corrupted audio from interrupting training.

The largest runtime cost was evaluation frequency rather than training itself.

## Conclusion

Exp048 establishes the first stable multilingual competition baseline.

Future experiments should reduce evaluation frequency while keeping multilingual training unchanged.

# Exp049 — Results

Reduce runtime by lowering evaluation/checkpoint frequency while keeping the Exp048 multilingual setup unchanged.

## Change

## Findings
Reducing evaluation/checkpoint cadence produced an approximately 80% runtime reduction with no degradation in WER.

## Conclusion
Exp049 becomes the new runtime-efficient multilingual baseline. Future experiments should inherit eval_steps=50 and save_steps=50 unless evaluation cadence is the variable being tested.
